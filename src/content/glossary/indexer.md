## 什麼是 Indexer？

Indexer (索引器) 是區塊鏈數據的中介層。最知名的項目是 **The Graph**。

它的功能是從區塊鏈上讀取原始數據，進行分類、整理、建立索引，並提供 API (通常是 GraphQL) 讓前端應用程式可以快速查詢。

## 大白話解釋

**Indexer 就是把「雜亂的倉庫」變成「有序的圖書館」。**

*   **沒有 Indexer:** 
    *   區塊鏈就像一個只會一直往後寫的流水帳日記本。如果你想問：「這個 NFT 以前的所有擁有者是誰？」，你必須從第一頁（創世區塊）開始，一頁一頁翻到最後一頁，這會慢到讓應用程式根本跑不動。
*   **有 Indexer:** 
    *   Indexer 先幫你把這些日記讀過一遍，做好了目錄和分類卡片。
    *   當你想問同樣的問題，Indexer 一秒鐘就能告訴你答案。

## 為什麼需要 Indexer？

對於 DApp (去中心化應用) 開發者來說，直接從區塊鏈節點 (Node) 讀取複雜數據是非常低效且昂貴的。
*   Uniswap 顯示的「歷史交易記錄」。
*   OpenSea 顯示的「某個 Collection 的所有 NFT」。
這些功能背後，幾乎都是靠 Indexer 在支撐。

## 運作原理

1.  **監聽:** Indexer 監聽區塊鏈上的事件 (Events)。
2.  **處理:** 根據預定義的腳本 (Subgraph)，解析這些事件。
3.  **儲存:** 將整理好的數據存入資料庫。
4.  **查詢:** 開發者通過 API 獲取數據。

## 總結

Indexer 是 Web3 堆棧中不可或缺的「中間件 (Middleware)」。雖然一般用戶感覺不到它的存在，但沒有它，我們現在使用的 DApp 速度會慢得像撥接上網。
