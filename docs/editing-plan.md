## 概要
這份企劃說明在 `src/pages/projects` 目前為 placeholder 的情況下，下一階段應該依照哪些視角與順序，逐步填補真實內容。目標是讓讀者從認識主流、理解基礎建設、連結新興趨勢與實用工具等層面慢慢深入。

## 優先順序與對應主題
1. **主流與學習入口**（比特幣、以太坊、Cardano）
   - 說明它們的市場定位、可代表的典型應用（如智能合約、穩定幣或高安全性支付），並列出重要節點（例如比特幣 ETF、Eth 2.0 演進、Cardano 的研究治理）。
   - 建議段落：導讀＋歷史／技術背景；關鍵指標（市值、手續費、網路效能）；新手推薦觀察的里程碑。
2. **擴容與基礎建設**（Layer-2 方案、擴容架構、混合 DAG）
   - 介紹目前最被討論的擴容路線（如 Rollup、zkEVM、BlockDAG、Bitcoin Hyper），說明它們解決的痛點與市場進展。
   - 段落大綱：為何需要擴容；不同流派特色；對應的代表產品/研究；開發者及資金動向。
3. **AI / DePIN / Web3 智能應用**（如 Ozak AI、MINION）
   - 將區塊鏈與 AI 或去中心化實體網路結合的故事拆成「技術要點」、「潛在用例」與「生態夥伴」三段。
   - 強調 2024-2026 的趨勢：AI 強化 oracle /預測、DePIN 資料蒐集與算力協調。
4. **使用者體驗與多鏈整合**（Best Wallet、Toncoin 生態）
   - 比較何種錢包與鏈最能降低新使用者門檻，並補充多鏈整合、原生質押、跨鏈交易速度等指標。
   - 段落：UX／WALLET 精選；Ton 的技術構成與可擴充性；整合案例與生態夥伴。
5. **實用支付與穩定性**（Litecoin、Lightning Network）
   - 針對希望避開高波動但仍想體驗加密的訪客，說明穩定性與手續費降低的策略。
   - 段落：貨幣定位；網路實際運作；近期升級（如擴容、LN 進展）；與主流資產的區隔。
6. **社群與 meme 經濟**（Shiba Inu、LilPEPE）
   - 展示 meme 幣如何從社群動員轉為有建設性項目（例如 Layer-2、DAO治理、NFT/遊戲）。
   - 段落：社群文化；效用化路線；里程碑（Shibarium、LilPEPE 生態）；風險與觀察指標。

## 段落結構建議
- **引言**：用簡短段落說明本頁用途（介紹不同類型加密項目）與閱讀順序。
- **主題區塊**：每個區塊包含「標題」→「短描述」→「示意資料 / 狀態表」（可放 bullet 或小表）→「延伸連結」。
- **取訊與觀察**：用小段列出「Room for research」與「值得追蹤的指標」，讓 placeholder 內容也有明確補完方向。
- **視覺延伸建議**：可搭配 icon、時間線或情境圖（例如：從主流到 meme 的想像旅程）。
